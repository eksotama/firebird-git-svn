Einführung
==========

Willkommen zu Firebird 2.1.


Verwendungszweck
================

Dieser Release Candidate ist für erfahrene Firebird Benutzer
bestimmt. Dieser wurde angemessen getestet und alle neue
Features sollten nun entsprechend dokumentiert sein.

Verwenden Sie dieses Release nicht in Produktionsumgebungen.


Bekannte Probleme
=================

Windows Vista - Die Setuproutine kann Firebird nicht als
Dienst starten, jedoch wird der Dienst nach einem Neustart
korrekt gestartet. Als Workaround können Sie auf dem Firebird
Systemsteuerungs-Applet mit einem SHIFT-Rechtsklick 'Ausführen
als...' auswählen und den Dienst von dort aus starten.


Features in diesem Release
==========================

Nähere Informationen dazu finden Sie in den Release Notes.


Installation
============

Die Installationsthemen werden im Detail in der Installations-
anleitung, die nach der Installation im doc Verzeichnis verfügbar
ist, diskutiert. Eine kurze Zusammenfassung dieser Themen finden
Sie auch in der installation_liesmich.txt. Diese Datei finden
Sie ebenfalls im doc Verzeichnis nach der Installation.


Bugs berichten
==============

Bevor Sie einen Bug berichten:

 o Stellen Sie sicher, dass Sie verstehen, wie Firebird arbeitet.
   Womöglich handelt es sich um gar kein Problem.

 o Vielleicht hat jemand bereits das selbe Problem berichtet? Sie
   können existierende Problemberichte hier einsehen:
   http://tracker.firebirdsql.org/secure/BrowseProject.jspa

 o Falls Sie sich nicht sicher sind, diskutieren Sie das
   Problem am besten vorab in der Firebird-devel Liste.
   Dieser Liste können Sie hier beitreten:

	http://lists.sourceforge.net/lists/listinfo/firebird-devel

   Diese Liste ist auch über ein Newsgroup-Interface hier
   einsehbar:

	news://news.atkin.com/


Verwenden Sie bitte die Firebird-devel Liste nicht für
technische Supportanfragen, solange es sich bei Ihrer
Anfrage nicht um ein spezifisches Firebird 2.1 Feature
handelt.


Happy Testing!

Ihr Firebird Team.
