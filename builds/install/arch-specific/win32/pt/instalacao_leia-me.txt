Servidor SQL Firebird 2.0 ALFA 3
==================================================

  ** IMPORTANTE **

  A ODS foi modificada. Consulte as notas abaixo 
  relativas à nova ODS e instalação sobre versões
  anteriores do Firebird.

  ** ********* **

Este documento constitui um guia para instalar o kit do
Firebird 2.0 Alpha 3 na plataforma Win32. Estas notas 
referem-se ao próprio kit de instalação, em vez do 
Firebird 2.0 em geral.

Antes de Instalar
-------------------

Recomenda-se que REMOVA quaisquer versões prévias do 
Firebird ou InterBase antes de instalar este kit.
É também importante verificar se o fbclient.dll e o
gds32.dll são removidos do <system32>.


Nova ODS 11.0
------------

Alfa 3 introduz uma nova ODS 11.0 a qual é totalmente
incompatível com a ODS 11.0 usada por versões alfa 
anteriores.

Se está a instalar a Alfa 3 pela primeira vez e já 
possui um Firebird 2 alfa instalado, recomenda-se que
proceda à instalação sobre uma pasta vazia.
Tal deve-se à remoção de uma instalação preservar
a base de dados existente security2. E como essa 
versão anterior usa uma ODS incompatível, isso impedirá
que consiga estabelecer ligação com o servidor.


Instalação em sistemas com o Firebird 1.5
-----------------------------------------

A base de dados de segurança foi modificada. A nova 
base de dados foi renomeada para security2.fdb e usa um
algoritmo reforçado de encriptação de palavras-passes. 
Poderá encontrar mais informações acerca das novas
funcionalidades de segurança em doc/README.sha1.txt.

A nova estrutura da base de dados e a nova encriptação
apresentam algumas dificuldades de migração para sítios
que possuam muitos id de utilizador. 
O programa de instalação não efectua qualquer tentativa 
de migração.  Para a Alfa 1 tal terá que ser efectuado 
manualmente. Uma técnica possível de migração encontra-
-se disponível em misc/upgrade/security/. Uma versão 
futura de teste tratará esta questão da migração de 
modo mais completo.


Instalação em sistemas com o Firebird 2.0
-----------------------------------------

Além da modificação da ODS há a considerar que o 
processo de instalação não consegue detectar se um 
servidor de uma versão Firebird 2.0 Alfa 1,2 ou 3 se 
encontra activo como um serviço. Como tal não receberá
qualquer aviso até que o programa de instalação tente 
sobrepor um ficheiro activo.


Novas funcionalidades de instalação
-----------------------------------

o Se um firebird.conf anterior for detectado o programa
  de instalação instalará o ficheiro conf por omissão 
  como:

  firebird.conf.default.

  O ficheiro firebird.conf permanecerá inalterado.
  O objectivo é garantir que as restrições de acesso se
  mantenham após a nova instalação.


o Foi adicionado ao programa de instalação suporte para 
  diversas linguagens. Este suporte SÓ foi implementado
  para o processo de instalação.


Problemas de instalação conhecidos
----------------------------------

Uma instalação apenas do Cliente encontra-se de facto 
a instalar o executável do servidor. Tal será corrigido
em versões beta posteriores. Entretanto poderá apagar o 
ficheiro .exe. Isto poderá ser automatizado através de 
um script de instalação.


Remoção
-------

o É preferível que este kit seja removido correctamente
  usando a aplicação de remoção fornecida. Tal poderá
  ser efectuado recorrendo ao Painel de Controlo.
  Alternativamente poderá ser removido executando o
  unins000.exe directamente a partir da pasta de
  instalação.
  
o Se o Firebird se encontrar em execução como aplicação
  (em vez de um serviço) recomenda-se que o interrompa
  manualmente antes de proceder à remoção. O motivo é 
  porque o programa de remoção não pode parar uma 
  aplicação em execução. Se um servidor se encontrar em
  execução durante a remoção o processo terminará com 
  erros.
  Terá que remover o restante manualmente.

o A remoção não apaga os seguintes ficheiros da pasta
  de instalação:

  - aliases.conf
  - firebird.conf
  - firebird.log
  - security2.fdb

  Tal é intencional. Estes ficheiros são potencialmente
  modificáveis pelos utilizadores e poderão ser 
  necessários caso o Firebird volte a ser reinstalado. 
  Poderão contudo ser apagados manualmente caso não 
  pretenda usá-los mais.


Outras Notas
------------

  O Firebird requer o WinSock2. Todas as plataformas 
  Win32 deverão possui-lo, excepto para o Win95. 
  Um teste sobre a sua presença é feito durante o pro-
  cesso de instalação.
  Se não for encontrado a instalação fracassará. Poderá
  visitar este link:

    http://support.microsoft.com/default.aspx?scid=kb;EN-US;q177719

  para obter informação sobre a sua instalação.
